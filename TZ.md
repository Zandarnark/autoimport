# Техническое задание
## Дипломный проект: Сайт параллельного импорта автомобилей и автозапчастей

---

## 1. Общее описание проекта

Веб-приложение для заказа автомобилей и автозапчастей из-за границы по схеме параллельного импорта. Платформа объединяет каталог транспортных средств и деталей с возможностью оформления заявки на доставку, расчётом стоимости и отслеживанием статуса заказа.

---

## 2. Роли пользователей

| Роль | Описание |
|------|----------|
| **Гость** | Просмотр каталогов, поиска, карточек товаров. Регистрация/авторизация. Оформление заявки невозможно. |
| **Пользователь** | Всё что у гостя + оформление заявок, личный кабинет, история заказов, отслеживание статуса, избранное, уведомления. |
| **Администратор** | Всё что у пользователя + панель управления: CRUD каталогов, управление заявками (смена статуса, комментарии), управление пользователями, просмотр аналитики. |

---

## 3. Функциональные требования

### 3.1. Регистрация и авторизация
- Регистрация по email + пароль (с подтверждением email)
- Авторизация (login/logout)
- Восстановление пароля
- Профиль пользователя: ФИО, телефон, email, адрес доставки
- Редактирование профиля

### 3.2. Каталог автомобилей
- Список автомобилей с фильтрацией:
  - Бренд, модель, год выпуска
  - Тип кузова, тип двигателя, объём двигателя
  - Страна происхождения
  - Ценовой диапазон
  - Пробег
- Карточка автомобиля:
  - Фотогалерея (карусель)
  - Характеристики
  - Комплектация
  - Страна происхождения
  - Расчётная стоимость с учётом пошлин и доставки
  - Кнопка «Оформить заявку»
  - Кнопка «В избранное»

### 3.3. Каталог автозапчастей
- Список запчастей с фильтрацией:
  - Категория (двигатель, подвеска, кузов, электрика, салон, расходники и т.д.)
  - Бренд детали, артикул
  - Совместимость (по марке/модели авто)
  - Страна происхождения
  - Ценовой диапазон
  - Наличие на складе
- Карточка запчасти:
  - Фото
  - Характеристики, артикул
  - Совместимость
  - Расчётная стоимость с доставкой
  - Кнопка «Оформить заявку»
  - Кнопка «В избранное»

### 3.4. Калькулятор стоимости
- Выбор страны отправления
- Выбор типа транспорта/детали
- Автоматический расчёт:
  - Стоимость товара
  - Таможенные пошлины
  - Логистика (доставка)
  - Услуги брокера
  - Комиссия сервиса
  - **Итого**
- Возможность ручной корректировки (для админа)

### 3.5. Оформление заявки
- Выбор товара из каталога или произвольный запрос (свободная форма)
- Заполнение данных доставки
- Предварительный расчёт стоимости
- Подтверждение заявки
- Заглушка оплаты (см. п. 3.9)

### 3.6. Личный кабинет пользователя
- Список заявок со статусами:
  - Новая → В обработке → Согласовано → В доставке → Получено → Завершено
  - Возможные: Отклонено, Требует уточнения
- Детали заявки: статус, история изменений, комментарии менеджера
- Избранные товары
- Настройки профиля
- Уведомления (смена статуса заявки)

### 3.7. Панель администратора
- **Управление каталогами**: добавление/редактирование/удаление автомобилей и запчастей
- **Управление заявками**: просмотр всех заявок, смена статуса, добавление комментариев для пользователя, назначение менеджера
- **Управление пользователями**: список, блокировка, смена роли
- **Аналитика**: количество заявок по статусам, популярные бренды, динамика за период (графики)
- **Настройки калькулятора**: редактирование тарифов пошлин, логистики, комиссий

### 3.8. Дополнительный функционал
- **Поиск**: глобальный поиск по каталогам с автодополнением
- **Сравнение**: сравнение до 3 автомобилей бок о бок по характеристикам
- **Запрос на подбор**: форма «Не нашли нужное?» — пользователь описывает что нужно, заявка уходит админу
- **Блог/Новости**: раздел с новостями параллельного импорта, изменениях в законодательстве (управляется админом)
- **Чат/Сообщения**: обмен сообщениями между пользователем и менеджером внутри заявки
- **Отзывы**: после завершения заказа пользователь может оставить отзыв

### 3.9. Заглушка оплаты
- При нажатии «Оплатить» отображается модальное окно:
  - Имитация формы оплаты (номер карты, срок, CVV — **не обрабатываются реально**)
  - Кнопка «Оплатить» с имитацией загрузки (2-3 секунды)
  - После «оплаты» — сообщение «Оплата прошла успешно (тестовый режим)»
  - Статус заявки меняется на «Оплачено»
- Везде где фигурирует оплата — плашка «Демо-режим. Реальная оплата не производится.»
- Никакие платёжные системы не интегрируются

## 4. Нефункциональные требования

### 4.1. UI/UX и анимации
- **Переходы между страницами**: плавный fade/slide (Framer Motion / CSS transitions)
- **Появление элементов**: при скролле — плавное появление (fade-in, slide-up) с небольшой задержкой между элементами
- **Hover-эффекты**: карточки товаров — лёгкий подъём (translateY) + тень, кнопки — плавная смена цвета/масштаба
- **Модальные окна**: плавное открытие/закрытие (scale + fade), оверлей с blur
- **Скелетон-загрузка**: вместо спиннеров — skeleton-плейсхолдеры при загрузке данных
- **Уведомления (toast)**: плавное появление сверху/снизу с автозакрытием
- **Карусель фото**: плавная смена слайдов с плавной анимацией
- **Фильтры**: плавное раскрытие/сворачивание, плавное обновление списка результатов
- **Статус-бар заявок**: анимированная прогресс-линия по этапам доставки

### 4.2. Адаптивность
- Mobile-first подход
- Корректное отображение на экранах от 320px до 1920px+
- Бургер-меню на мобильных

### 4.3. Производительность
- Ленивая загрузка изображений (lazy loading)
- Пагинация / бесконечный скролл для каталогов
- Оптимизация бандла (code splitting)

## 6. Структура базы данных (основные сущности)

```
User
├── id, email, passwordHash, firstName, lastName, phone, role, createdAt, updatedAt

Car
├── id, brand, model, year, bodyType, engineType, engineVolume, mileage, country, price, images[], description, specifications, createdAt, updatedAt

Part
├── id, name, category, brand, article, compatibility[], country, price, inStock, images[], description, specifications, createdAt, updatedAt

Order
├── id, userId, type (car|part|custom), itemId (nullable), customDescription, status, totalPrice, deliveryAddress, comments[], managerId, createdAt, updatedAt

OrderStatusHistory
├── id, orderId, status, comment, changedBy, createdAt

Favorite
├── id, userId, itemType (car|part), itemId, createdAt

Message
├── id, orderId, senderId, text, createdAt

Review
├── id, userId, orderId, rating, text, createdAt

News
├── id, title, content, image, authorId, createdAt, updatedAt

CalculatorSettings
├── id, country, dutyRate, logisticsCost, brokerFee, commissionRate
```

- Иконки: Lucide React, линейный стиль
- Иллюстрации: абстрактные геометрические элементы на фоне hero
